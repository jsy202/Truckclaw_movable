 1. 시뮬레이션 환경 준비
  먼저 CARLA 시뮬레이터가 실행 중이고 Town06 (고속도로 맵)이 로드되어 있어야 합니다. (이미 플레이 중이라고 하셨으므로 다음 단계로
  진행하시면 됩니다.)

  2. 브리지 서버(Bridge Server) 실행
  에이전트들 사이의 협상 상태를 관리하는 뇌 역할을 합니다. 터미널에서 실행하세요.

   1 python3 /home/user/truck-platoon-openclaw-improve/bridge/platoon_bridge_server.py

  3. CARLA 시나리오 실행
  실제 트럭들이 도로 위에서 움직이는 물리 시뮬레이션입니다. 별도의 터미널에서 실행하세요.

   1 # CARLA PythonAPI 경로 설정 후 실행
   2 export PYTHONPATH=$PYTHONPATH:/home/user/carla_source/PythonAPI/carla
   3 python3 /home/user/truck-platoon-openclaw-improve/scenario/examples/two_platoon_truck_scenario.py
   * 주의: 현재 테스트를 위해 시작 5초 후 자동으로 중간 차량(truck1)이 이송되도록 설정해 두었습니다. 가만히 두시면 알아서 거리를 벌리고
     합류하는 것을 볼 수 있습니다.

  4. 차량 목적지 수정 및 실시간 반영 (커스텀 테스트)
  만약 다른 차량을 옮기고 싶다면 다음과 같이 하세요.

   1. /home/user/truck-platoon-openclaw-improve/config/platoons.json 파일을 엽니다.
   2. 이송하고 싶은 차량의 destination_id를 상대 소대의 목적지(dest_b 또는 dest_a)와 일치시킵니다.
   3. 브리지 서버가 켜진 상태에서 아래 명령을 입력하여 설정을 새로고침합니다.

   1     python3 /home/user/truck-platoon-openclaw-improve/bridge/platoon_bridge_ctl.py reload
   4. 이제 시나리오를 다시 실행하거나, Discord 에이전트(OpenClaw)에게 말을 걸어 협상을 시작하면 바뀐 목적지대로 움직입니다.

  5. 수동 제어 (CLI 활용)
  자동 이송 대신 직접 명령을 내리고 싶다면, 시나리오 실행 중 다른 터미널에서 아래 명령을 사용하세요.

   1 # 특정 차량 이송 요청 생성 (예: platoon_a의 truck1을 platoon_b로)
   2 python3 /home/user/truck-platoon-openclaw-improve/bridge/platoon_bridge_ctl.py request platoon_a_truck1 platoon_a platoon_b
   3
   4 # 생성된 request_id 수락 (snapshot 명령으로 ID 확인 가능)
   5 python3 /home/user/truck-platoon-openclaw-improve/bridge/platoon_bridge_ctl.py accept <request_id>
   6
   7 # 최종 커밋 (이 순간 CARLA에서 차량이 움직이기 시작합니다)
   8 python3 /home/user/truck-platoon-openclaw-improve/bridge/platoon_bridge_ctl.py commit <request_id>

  💡 팁: 현재 시나리오의 특징
   * 화면 렌더링: --no-render 옵션을 빼고 실행하면 트럭들이 움직이는 모습을 직접 볼 수 있습니다.
   * 자동 종료: 현재 코드는 합류가 완료되면 즉시 종료되도록 되어 있습니다. 계속 주행하게 두고 싶으시면 제가 종료 로직을 제거해 드릴 수
     있습니다.
